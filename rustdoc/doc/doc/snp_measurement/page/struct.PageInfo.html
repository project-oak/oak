Expand description

Implementation of the Page Info structure used for extending the measurement in each step.

See table 67 in https://www.amd.com/system/files/TechDocs/56860.pdf.

Fields§

§digest_cur: [u8; 48]

The current measurement up to this point.

§contents: [u8; 48]

The SHA-384 digest of the contents to be measured for normal and VMSA pages, or all zeros for all other page types.

§_length: u16

The length of this struct in bytes.

§page_type: PageType

The type of page being measured.

§_imi_page: ImiPage

Whether the page is part of an initial migration image. For now we treat this as reserved and zero.

§_reserved: u8

Reserved. Must be 0.

§_vmpl1_perms: u8

The permissions for VMPL1. For now we treat this as reserved and zero.

§_vmpl2_perms: u8

The permissions for VMPL2. For now we treat this as reserved and zero.

§_vmpl3_perms: u8

The permissions for VMPL3. For now we treat this as reserved and zero.

§gpa: u64

The guest-physical address of the page being measured.

fn set_contents_from_page_bytes(&mut self, page_bytes: &[u8])

Sets the contents field based to the SHA-384 digest of the byte contents of a 4KiB memory page.

If fewer than 4KiB of data is received the page is padded with zeros to fill the entire 4KiB area.
