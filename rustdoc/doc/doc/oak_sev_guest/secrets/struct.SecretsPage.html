Expand description

Representation of the secrets page.

See: Table 68 in https://www.amd.com/system/files/TechDocs/56860.pdf

Fields§

§version: u32

The version of the secrets page.

§imi_en: u32

The least significant bit indicates whether an initial migration image is enabled in the guest context. All other bits are reserved and must be zero.

Use SecretsPage::get_imi_en to try to get this as an Imi enum.

§fms: u32

The family, model and stepping of the CPU as reported in CPUID Fn0000_0001_EAX. See https://en.wikipedia.org/wiki/CPUID#EAX=1:_Processor_Info_and_Feature_Bits.

§gosv: [u8; 16]

Guest-OS-visible workarounds provided by the hypervisor during SNP_LAUNCH_START. The format is hypervisor-defined.

§vmpck_0: [u8; 32]

VM-platform communication key 0. AES key used for encrypting messages to the platform.

§vmpck_1: [u8; 32]

VM-platform communication key 1. AES key used for encrypting messages to the platform.

§vmpck_2: [u8; 32]

VM-platform communication key 2. AES key used for encrypting messages to the platform.

§vmpck_3: [u8; 32]

VM-platform communication key 3. AES key used for encrypting messages to the platform.

§guest_area_0: GuestReservedArea

Area reserved for guest OS use.

§vmsa_tweak_bitmap: [u8; 64]

Bitmap indicating which quadwords of the VM Save Area have been tweaked. This is only used if the VMSA Register Protection feature is enabled.

§guest_area_1: [u8; 32]

Area reserved for guest OS use.

§tsc_factor: u32

Scaling factor that can be used for calculating the real CPU frequency.
